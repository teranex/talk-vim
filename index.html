Modes

Vim is a modal editor.

Modes

  • Insert mode
  • Normal mode
  • Visual mode
  • Command-line mode
  • Operator-pending mode
  • and a few other...

Insert mode

This is the mode where you actually type text. It has a few shortucts, such as C-W to remove the word before the cursor

Normal mode

This is the mode you will use the most. Every key, or combination of keys, is a special command.

Visual mode

Visual mode is an improvement of Vim over the old Vi. It let's you visually select text to act upon.

In fact there are 3 different visual modes

'Regular' visual mode

  • Let's you select text
  • Started by pressing v in normal mode

Linewise visual mode

  • Selects entire lines
  • Started with V (capital v) from normal mode or visual mode

Blockwise visual mode

  • Selects a block of text
  • Started with C-V from normal mode or visual mode
  • This can be very useful. Try, for example, I followed by some text and Esc

Visual mode

  • When in normal mode, you can always use gv to reselect the previously selected text in visual mode
  • Jump to the other end by pressing o

Command-line mode

Started with : to enter a command.
When searching (/) you are also in Command-line mode.

If you ever end up in Ex-mode by pressing Q or gQ simply type vi<enter> and carry-on.

Operator-pending mode

This is a very temporary mode which you enter when Vim is waiting for further input. For example after pressing d Vim is in Operator-pending mode while waiting for a motion.

Motions

Motions are how you move the cursor in Vim.

Jumping to another line

  • G: jump to last line
  • gg: jump to first line
  • 30gg: jump to line 30
### Relative jumps * `3j`: jump 3 lines down * `12k`: jump 12 lines up
### Moving by words * `e`: move to the end of the word * `E`: move to the end of the WORD * `w`: move to the next word * `W`: move to the next WORD * `b`: move back a word * `B`: move back a WORD
Wait a minute mister... a word, a WORD, ...WHAT?? * A **word** is a regular word, such as `javascript` * A **WORD** is everything until the next whitespace, such as `xml:foobar`
### Search inside the same line * `f`, `F` ``: jump to (_find_) the next / previous `` in the line * `t`, `T` ``: jump to before (_until_) the next / previous `` in the line * `;`: repeat * `,`: repeat on opposite direction
### search * A search (`/`) in vim is also a motion. * Repeating a search with `n` or `N` is also a search --- **Tip**: a very useful search command is `*`. It searches for the word under the cursor (and `#` to search backwards).
## Counts Most motions take a count. * `5f*`: find the fifth occurrence of ***** on the current line * `3b`: move 3 words back * `10j`: jump 10 lines down * `2n`: jump to the second next search result (skipping the immediate next)
## Commands
**Commands** are how you tell Vim to **do something**.
### Some commands * `d`: delete text * `c`: change text (delete and go to insert mode) * `r`: replace a single character * `v`: go to Visual mode * `~`: toggle case of character (related `gU` and `gu`) * `y`: yank (copy) text * `p`: paste text * `u`: undo an edit
A lot of **commands** can be **combined** with a **motion**. This is the real power of Vim. Most commands also take a **count**. `[count] Command [count] Motion`
### Actually deleting something When you use the `d` command, Vim expects you to tell it what you actually want to delete
### Actually deleting something * `dd`: simply delete the current line (linewise) * `d3j`: delete current and next 3 lines (linewise) * `dtx`: delete until the next **x** on the current line * `d2f(`: delete until the second occurance of **(**, including the **(** itself. * `d/foo`: delete until the next occurance of **foo**
# text-objects
Text objects are **smarter** methods to tell Vim which **piece of text** you want to use, without manually having go to the start and the end of the text.
Most text objects have an `i` and an `a` variant. * the `i`-variant selects _inside_ the text-object * the `a`-variant selects _around_ the text-object
## some examples * `viw`: visually select the current word * `di"`: delete everything inside the quotes * `ca(`: change everything inside parentheses, including the parentheses themselves.
## But there is more!
This entire system is extensible. Plugins and mappings can add new motions, commands and text-objects which can be combined just like the built-in stuff.
### An example * The **tcomment** plugin adds the `gc` command to toggle comments * The **indent-text object** plugin adds the `ii` text-object to select everything with the same indent ``` if ($foo) { doSomething(); thenDoSomethingElse(); } ``` With your cursor on doSomething, press `gcii` to comment everything inside the if-block.

The dot-command

Repeats the last edit.
## The dot-command A useful example: * delete the current line: `dd` * delete the next line: `.` * and again: `.` * oops, didn't want to delete that last line: `u`
## The dot-command Another useful example: * search for **bar** by typing `/bar` * change it to foo by typing `cefoo` * jump to the next match: `n` * also replace: `.` * repeat..
# Marks
## Regular marks Using marks, you can mark a location in a text file for later reference. * lower-case marks are specific to a buffer * upper-case marks are global
## Marking a location To create a mark simply press `ma` to set mark **a**, `mE` to set mark **E** and so on.
## Jumping to a mark * `'a`: using **'** followed by a mark will jump to the **beginning** of the line where that mark was placed. * ``a`: using **`** followed by a mark will jump to the **exact location** where that mark was placed. --- These commands are also motions and can be combined with commands such as `d` and `c`.
## Automatic marks Vim also automatically creates marks for you * `1-9`: most recently used files * < & >: beginning and end of the current or most recent visual block * `[` & `]`: beginning and end of last modified block (for example from a paste) * `'`: previous location (before a jump) * `.`: position where the last change was made * and a few other
## Back to visual mode What actually happens in Visual mode? You tell Vim to set the **<** and **>** marks and then act on the region of text between those marks. Try pressing `:` with an active visual selection: you get **:'<,'>**. This specifies the range for the command to be the visual selection
# Ranges
When executing a command such as `:sort`, you can specify a range to act upon: `:12,18sort`.
* `:5,30`: from line 5 to line 30 * `:.,$`: from the current line (`.`) to the end of the file (`$`) * `:%`: the entire file (this is a shorthand for `0,$`) * `:'a,'b`: from mark **a** to mark **b** * `:-5,+12`: 5 lines before the cursor to 12 lines after the cursor
# Macros
While the **dot-command** can be very useful and should be used as much as possible, sometimes you will want to **repeatedly** perform a **more complex command**
## Recording a macro To record a macro: * `qa`: start recording into **register a** * `[commands]`: perform all the necessary commands. This can be anything you would normally do * `q`: press `q` to stop recording the macro
## Registers you say?? Instead of just a single clipboard Vim uses registers for yank/paste, macros and a few other things.
### Different registers * `a-z`: Are the regular registers you can freely use * `A-Z`: Are the same as *a-z* but when yanking text it is appended * `0-9`: 0 = most recent yank, 1-9 previous deletes * `"`: the unnamed register. This is the default when no register is specified * `+` & `*`: system clipboards * `_`: blackhole register * `.`: last inserted text * check `:help registers` for a few other and more information
## Back to macros!
## Executing a macro `@a`: **execute** the macro which was recorded in **register a** * This also takes a count * Quickly repeat last macro with `@@` * Macros can even execute themselves! `qxj@xq@x`: this will keep going down!
# Formatting text A quick intermezzo on how to format text
## Indentation * Set the correct indent settings (see [Vimcasts 2](http://vimcasts.org/episodes/tabs-and-spaces/)) * Then use the `=` command to let Vim correctly indent the specified text. * `==`: current line * `=ip`: current paragraph * `gg=G`: the entire file

Change Indentation

  • >: add more indentation
  • <: remove Indentation

Example: >3j indents 4 lines.
Then use . to indent more.

# Configuring vim
Vim is **extremely configurable**. Customizing Vim is nice. But there is also the chance that it becomes completely different from a regular 'Vim'.
When you want to start configuring Vim, it can be very interesting to have a look at other peoples configs. But don't just use somebody else's config! A good start is the [Vim-sensible](https://github.com/tpope/vim-sensible) plugin by Tim Pope.

Options

There are just to many options to even start explaining them.

  • get the current value: set listchars?
  • boolean options: set expandtab, set noexpandtab, set expandtab!
  • values: set tabstop=4
  • lists: set guioptions+=T, set guioptions-=m
## Keyboard mappings Mapping your own custom keybindings is another very powerful method to configure Vim. * `map`, `noremap`: map in most cases * `nmap`, `nnoremap`: map in normal mode * `vmap`, `vnoremap`: map in visual mode * `imap`, `inoremap`: map in insert mode * `cmap`, `cnoremap`: map in command-line mode * and a few more..
## Plugins Using plugins you can configure Vim even further and make it do really crazy stuff. * **ctrlp**: quickly open files by typing parts of their name * **fugitive + gitv**: awesome Git intergration * **syntastic**: automatic syntax checking when saving * **ultisnips**: snippets system * **vimwiki**: entire wiki inside vim
## Managing plugins If you are going to use more then 2 plugins, do yourself a favour and use a plugin manager * **pathogen**: my preference, clean and simple * **Vundle**: downloads plugins for you * **vim-addon-manager**: downloads plugins for you and has dependency-management
